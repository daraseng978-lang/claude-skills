# Operations and Delivery

Standard operating procedures, QA gates, and escalation thresholds.

## Onboarding (Days 0–7)

### Day 0 (sale closes)

Automated by Head agent:
- Stripe subscription created.
- Welcome email sent with calendar link to topic-strategy call.
- Customer record created in CRM with all sales-call notes attached.
- Slack (or shared Google Doc) workspace provisioned.

### Day 1–3

Founder (or human CSM at scale) + Deliver agent run a 60-minute topic strategy call:
- Confirm business goals, 3 top priority topics.
- Capture brand voice references (3 existing articles they love or aspire to).
- Collect CMS credentials (WordPress app password, Ghost admin key, Webflow token).
- Map target keyword universe using Brave Search + customer's Search Console (via OAuth).

### Day 4–7

- Deliver agent ships keyword-cluster strategy document.
- Customer approves or requests changes.
- First article drafted, QA passed, delivered by Day 7.

SLA: First article within 7 calendar days of subscription start. Missing this predicts 3x higher churn.

## Per-Article Production Loop

```
1. Keyword pick       (Deliver agent, from approved cluster)
2. SERP analysis      (Claude analyzes top 10 results for intent, length, format)
3. Outline            (H2/H3 + evidence-per-section map)
4. Source gathering   (Brave Search + scraping, 10–20 sources ranked)
5. Draft              (Opus 4.7, brand voice prompt + outline + sources)
6. QA Gate A: Facts   (every claim → cited URL, Haiku verifies)
7. QA Gate B: Voice   (Haiku scores vs brand reference; < 0.8 similarity rejects)
8. QA Gate C: SEO     (checklist: title, meta, H1, internal links, alt text, schema)
9. QA Gate D: Length  (±20% of target word count; longer if SERP median is longer)
10. Image selection   (Unsplash API, alt text generated, license logged)
11. Publish as draft  (CMS API, "pending review" state)
12. Customer notify   (email: "draft ready, here's the link")
13. Revision cycle    (tier-limited; agent never promises unlimited)
14. Publish live      (customer approves; agent flips to public)
15. Internal linking  (agent re-audits last 10 articles, adds links to new piece)
16. Submit to Search Console
```

## QA Gates in Detail

### Gate A: Facts
- Every statistic cites a URL published within 24 months.
- Named people referenced → verified exists at claimed role.
- No claims about regulated topics (medical, legal, financial advice) without Founder approval.
- Plagiarism score (Copyscape or equivalent) < 5% match to any source.

### Gate B: Voice
- 3 reference articles → embedding similarity baseline.
- Draft must score within 0.15 of baseline on tone, vocabulary, sentence rhythm.
- Failing drafts auto-revised up to 2x before escalation.

### Gate C: SEO checklist (binary)
- [ ] Focus keyword in title, H1, first 100 words, meta description
- [ ] 3+ H2 sections, at least 1 H3 per H2
- [ ] Internal links to 3+ customer articles (if available)
- [ ] External links to 3+ authoritative sources
- [ ] Meta description 140–160 chars
- [ ] Featured image with descriptive alt text
- [ ] Schema markup (Article or HowTo)
- [ ] Clear CTA at bottom

### Gate D: Length
- Match or exceed the SERP median length for top 10.
- Never below 1,200 words (signals thin content).
- Never above 4,000 words unless pillar page.

## Support Tickets

Inbound customer requests are triaged by the Head agent within 1 business hour:

| Type | Handled by | SLA |
|---|---|---|
| "Revise this article" | Deliver agent | 48 hours |
| "Change our brand voice" | Deliver + Founder approval | 5 days |
| "Add a new topic cluster" | Deliver agent | 3 days |
| "Billing question" | Finance agent | 1 business day |
| "Cancel" | Head agent → Founder | 4 hours |
| "I need to talk to a human" | **Founder** | Same day |
| "Legal/compliance question" | **Founder** | Same day |

Agent responses to support never say "as an AI" — say "our team." The customer knows the service is AI-delivered from the sales page; they don't need reminders in every email.

## Monthly Customer Review

Retain agent generates a monthly report per customer:
- Articles delivered vs tier limit
- Keywords gained/lost in Search Console
- Traffic and conversion deltas
- Top-performing article
- Next month's plan + 5 proposed topics

Delivered on the 3rd of each month. Customer has 5 days to push back; otherwise next month's cluster auto-approves.

## QBR (Quarterly Business Review)

Every 3 months:
- 45-minute video call (Zoom / Google Meet).
- **Human-run** (founder or CSM). Not an agent call.
- Deck auto-generated by Retain agent with 90-day performance narrative.
- Renewal / expansion conversation if applicable.

Annual customers: QBR happens at months 3, 6, 9, 12. Month 12 is the renewal conversation.

## Offboarding

When a customer cancels:
1. Head agent confirms cancellation date (end of paid period; no prorated refunds).
2. Finance agent stops auto-renew.
3. Retain agent runs a 15-min exit interview (founder attends if MRR > $4k).
4. All data exported to customer within 7 days.
5. CMS credentials revoked within 14 days.

Every cancellation becomes a line in the churn-reason log. Three cancellations with the same reason → product change discussion at next weekly review.

## Security and Compliance Basics

- Customer CMS credentials stored in a secrets vault (Doppler, 1Password, or HashiCorp Vault). Never in code or repo.
- Agent access to credentials is just-in-time, scoped to the specific job, and logged.
- Customer data retention: 90 days post-cancel, then purged.
- GDPR / CCPA: Data export and deletion requests honored within 30 days.
- Annual security review by founder (or external consultant at $100k+ MRR).
